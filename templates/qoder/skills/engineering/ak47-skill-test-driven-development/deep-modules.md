# 深度模块

来源: "A Philosophy of Software Design"

**深度模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   小接口            │  ← 方法少,参数简单
├─────────────────────┤
│                     │
│                     │
│  深实现             │  ← 复杂逻辑隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现 (避免)

```
┌─────────────────────────────────┐
│       大接口                    │  ← 方法多,参数复杂
├─────────────────────────────────┤
│  薄实现                         │  ← 只是传递
└─────────────────────────────────┘
```

设计接口时问:
- 能减少方法数量吗?
- 能简化参数吗?
- 能隐藏更多复杂性吗?
